# AI Teacher PoC — 데모 가이드

## 한 줄 요약

> **교사가 AI로 리포트, 시험, 교안을 만들고, 학생은 AI Tutor에서 과제를 풀면, 채점 결과가 다시 교사에게 돌아온다.**

---

## 이게 왜 필요한가?

| | 현재 교사 업무 | AI Teacher |
|---|---|---|
| 학생 파악 | 시험 후 성적표로 파악 | AI가 대화 패턴까지 실시간 분석 |
| 리포트 | 수동 작성 (30분/명) | AI 즉시 생성 (10초) |
| 시험 출제 | 수동 (3~4시간) | AI 자동 (10분, 약점 반영) |
| 교안 작성 | 수동 (1~2시간) | AI 자동 (5분, 반 약점 반영) |
| 채점 | 수동 (반나절) | 학생 제출 시 AI 즉시 채점 |
| 데이터 활용 | 시험 점수만 | 실수 유형, 학습 선호, 성장 추세 |

## 핵심 기능 5가지

### 1. 📊 학생 리포트 — AI가 학생을 분석한다
학생 한 명 선택 → 10초 후 종합 리포트 생성
- 강점/약점 분석 (Mem0 데이터 기반)
- 최근 학습 활동 요약 (대화 이력 20건 분석)
- 이전 리포트 대비 성장 추세
- 학부모 상담용 한줄 코멘트

### 2. 📝 시험 출제 — 약점 반영 맞춤 시험
단원 + 난이도 + 문제 수 입력 → 시험지 + 정답지 + 채점기준 자동 생성
- 학생 약점 반영 문제 포함 (선택)
- 이전 출제와 중복 방지
- 문제은행 30문제에서 유사 문제 참조

### 3. 📖 교안 생성 — 반 전체 약점 반영 수업 계획
단원 + 수업 시간 + 학생 선택 → 50분 수업 교안 자동 생성
- 4명 전원 Mem0 조회 → 공통 약점 도출
- 도입(5분) → 전개(30분) → 활동(10분) → 정리(5분)
- 수준별 활동 문제 분리 (상위/중위/하위)

### 4. 📋 과제 전달 + 현황 확인 — 교사-학생 연계
시험 출제 후 [과제로 전달] → 학생 AI Tutor에 자동 전달
- 과제 현황 대시보드에서 제출/채점 상태 확인
- 학생별 채점 결과 마크다운으로 상세 확인
- 채점 결과가 Mem0에 축적 → 다음 리포트에 자동 반영

### 5. 💬 AI Tutor 힌트 모드 — 학생이 스스로 풀게 돕는다
과제가 전달되면 AI Tutor가 자동으로 힌트 모드 전환
- "풀어줘" 해도 답 안 알려줌
- 단계별 힌트만 제공 (풀이 방향 → 구체적 방향 → 최후 수단)
- 개념 질문은 자유롭게 답변

## 추천 데모 시나리오

### 🎯 시나리오 1: 학생 리포트 생성 (2분)

```
1. AI Teacher → 교사 로그인
2. 📊 학생 리포트 메뉴
3. "📚 이중위 (중위, 60~80)" 선택
4. [리포트 생성] 클릭 (10초 대기)
5. 확인 포인트:
   - "부호 실수 반복" 약점 언급
   - "검산 생략" 패턴 언급
   - 이전 리포트 대비 성장 분석
```

### 🎯 시나리오 2: 시험 출제 + 과제 전달 (3분)

```
1. 📝 시험 출제 메뉴
2. 단원: "이차방정식", 문제 수: 5, 난이도: 중
3. 학생 약점 반영: "📚 이중위" 선택
4. [시험 출제] 클릭 (10초 대기)
5. 확인 포인트:
   - 시험지 / 정답지 / 채점기준 3탭
   - 이중위 약점(부호 처리) 관련 문제 포함
   - KaTeX 수식 렌더링 정상
6. [과제로 전달] 클릭
7. "4명에게 전달 완료" 확인
```

### 🎯 시나리오 3: 학생이 AI Tutor에서 과제 풀기 (5분)

```
1. AI Tutor → 이중위로 로그인
2. 확인: "📋 선생님이 내준 과제가 있어요!" 배너
3. 채팅에 아무 말이나 입력
4. 확인: AI가 "과제를 내줬어! 같이 풀어보자" 안내
5. "풀어줘" 입력 → AI가 답 안 알려줌 (힌트만)
6. "판별식이 뭐야?" → 개념 설명은 정상 제공
7. [과제 제출하기] 클릭 → 답안 입력 → [제출]
8. 자동 채점 결과 확인 (오답 분석 + 피드백)
```

### 🎯 시나리오 4: 교안 생성 (2분)

```
1. AI Teacher → 📖 교안 생성 메뉴
2. 단원: "이차방정식 판별식", 시간: 50분
3. 학생: 4명 전원 체크
4. [교안 생성] 클릭 (15초 대기)
5. 확인 포인트:
   - 반 약점 분석 (공통: 부호 처리)
   - 수준별 활동 분리 (상/중/하)
   - 도입→전개→활동→정리 구조
```

### 🎯 시나리오 5: 데이터 순환 확인 — 킬링 데모 (5분)

**이 시나리오가 PoC의 핵심 증명입니다.**

```
Step 1: 이중위 리포트 생성 → 내용 확인 (약점: 부호 실수)
Step 2: 시험 출제 → [과제로 전달]
Step 3: AI Tutor에서 이중위로 로그인 → 과제 풀기 → 일부러 부호 틀리게 제출
Step 4: 자동 채점 → "부호 실수 확인" + Mem0 축적
Step 5: AI Teacher에서 이중위 리포트 다시 생성
Step 6: 확인: "최근 시험에서도 부호 실수 확인됨" 반영!

핵심: 채점 → Mem0 축적 → 리포트에 즉시 반영 = 데이터 순환 증명
```

### 🎯 시나리오 6: 과제 현황 대시보드 (1분)

```
1. AI Teacher → 📋 과제 현황 메뉴
2. 과제 목록에서 최근 과제 클릭
3. 확인:
   - 학생별 상태 (✅완료 / ⏳미제출)
   - 채점 완료된 학생 [결과 보기] 클릭
   - 오답 분석 + 약점 연관 확인
```

## 기술 아키텍처 (비개발자용)

```
┌──────────────┐      과제 전달       ┌──────────────┐
│              │  ──────────────→     │              │
│  AI Teacher  │                      │  AI Tutor    │
│  (교사 앱)    │  ←──────────────    │  (학생 앱)    │
│              │      채점 결과        │              │
└──────┬───────┘                      └──────┬───────┘
       │                                     │
       │         ┌──────────────┐            │
       └────────→│  공유 두뇌    │←───────────┘
                 │              │
                 │ Mem0 (기억)  │
                 │ 문제은행     │
                 │ 대화 이력    │
                 └──────────────┘
```

**핵심 기술:**
- **Mem0:** 학생의 약점, 강점, 학습 패턴을 영구 기억. 대화할수록 정확해짐.
- **pgvector:** 30문제 문제은행에서 의미 기반 검색 (단순 키워드가 아닌 수학적 유사도).
- **n8n 워크플로우:** 5개 AI 프로세스를 코드 없이 시각적으로 관리.
- **IF 분기:** 과제 유무에 따라 AI 행동 자동 전환 (튜터↔감독관).

## FAQ

**Q: AI가 시험 답을 알려주면 어떡하나요?**
A: 과제 모드에서는 별도 Agent가 동작합니다. 시스템 프롬프트에 "절대 정답 알려주지 마"가 설정되어 있고, 힌트만 단계적으로 제공합니다.

**Q: 교사가 없어도 되는 건가요?**
A: 아닙니다. AI는 교사의 "보조 도구"입니다. 수업 진행, 학생 관리, 교육적 판단은 교사의 영역입니다. AI는 반복적인 업무(채점, 리포트 작성 등)를 대신합니다.

**Q: 데이터가 실시간으로 반영되나요?**
A: 네. 학생이 AI Tutor에서 과제를 풀면 즉시 채점되고, Mem0에 약점이 축적됩니다. 교사가 리포트를 생성하면 최신 데이터가 바로 반영됩니다.

**Q: 비용이 얼마나 드나요?**
A: PoC 기준 월 약 $20입니다. 핵심 인프라(n8n, Mem0)를 Self-hosted로 운영하여 비용을 최소화했습니다.

**Q: 수학 외 다른 과목도 가능한가요?**
A: 네. 현재는 수학 PoC이지만, 문제은행과 Agent 프롬프트만 교체하면 영어, 과학 등 확장 가능합니다. 영어는 대화형 학습 모드로 개발 예정입니다.

**Q: 학생 데이터 보안은?**
A: Mem0와 n8n을 자체 서버에서 운영하여 외부 유출이 없습니다. Supabase RLS 정책으로 본인 데이터만 접근 가능합니다.
